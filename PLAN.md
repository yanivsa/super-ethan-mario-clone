# Super Ethan Holiday Plan

## Vision
לבנות פלטפורמר 2D בהשראת Mario, אבל לא כ-clone גנרי: כל שלב מייצג חג יהודי אחר, עם רקע, mood, אויבים, יעד סיום ושפה מכנית שמרגישים כמו אותו חג.

הקו המנחה:
- טון משחקי, הרפתקני וצבעוני.
- כבוד למסורת, בלי קריקטורות זולות.
- כל חג משנה לא רק צבעים אלא גם מטרה, אווירה ותחושת משחק.
- מפת הקמפיין היא "מחזור השנה", לא אוסף של עולמות מנותקים.

## Campaign Arc
הגיבור נע בין חגי השנה משום שהמחזור התפרק, וכל חג איבד את המוטיב המרכזי שלו. בכל שלב מחזירים מוטיב אחר למקומו, עד שבסוף השנה כולה נסגרת מחדש.

סדר השלבים:
1. פסח
2. ראש השנה
3. חנוכה
4. ט"ו בשבט
5. סוכות
6. פורים
7. שבועות
8. שמחת תורה

## Holiday Worlds

### 1. פסח: דרך היציאה
- Setting: מצרים, גיזה, מדבר, שער יציאה.
- Visuals: לבני מצה, אבן חול, זהב מדברי, אווירת מרדף.
- Core mechanic: חיפזון, פלטפורמות מצה זמניות, מקטעי בריחה.
- Enemies: שומרי חצר פרעה, מרכבות קלות, סיירי ארמון.
- Boss / finale: פרעה בשער החירות.
- Goal set piece: שער יציאה ממצרים.

### 2. ראש השנה: שער השופר
- Setting: עיר שערים לבנה, אור בוקר, שוק דבש ורימונים.
- Visuals: זהב רך, לבן, דבש, אדום תפוח.
- Core mechanic: תקיעת שופר מפעילה שערים ופותחת חלונות זמן.
- Enemies: שומרי סף, ערפל, יצורי עיכוב.
- Boss / finale: שומר שערי השנה.
- Goal set piece: שער שופר גדול.

### 3. חנוכה: מבצר הנרות
- Setting: עיר חורפית ומקדש מואר.
- Visuals: כחול לילה, זהב, אש, אבני חורף.
- Core mechanic: אור מול חושך, הדלקת מוקדים שמייצבים מסלולים.
- Enemies: חוליות חושך, מכבי-אש עוינים, גלגלי חושך.
- Boss / finale: מפקד החושך.
- Goal set piece: מנורה מרכזית.

### 4. ט"ו בשבט: גינת הנבטים
- Setting: בוסתן, טרסות, נחלים ושורשים.
- Visuals: ירוקים רעננים, ורוד פריחה, חום אדמה.
- Core mechanic: צמיחה של פלטפורמות וענפים.
- Enemies: קוצים, חרקים, רוחות קור שמייבשות.
- Boss / finale: רוח כפור או שומר היער.
- Goal set piece: חורשה פורחת.

### 5. סוכות: עיר הסוכות
- Setting: יער סכך, סוכות תלויות, גשרים מקנים.
- Visuals: ירוק-צהוב, עץ, קישוטים ואורות.
- Core mechanic: מחסה נע ותנועה בתוך רוח וגשם.
- Enemies: רוחות סתיו, ענפים נופלים, חרקי יער.
- Boss / finale: סופת סכך.
- Goal set piece: סוכה גדולה ומוארת.

### 6. פורים: מגילת הצללים
- Setting: ארמון פרסי, גגות, מסכות, שוק חגיגי.
- Visuals: סגול, טורקיז, ורוד, זהב.
- Core mechanic: היפוכים, הסוואה, מסלולים מתחלפים.
- Enemies: שומרים מחופשים, שליחי גזירה, בובות חצר.
- Boss / finale: המן כדמות תיאטרלית-מאיימת.
- Goal set piece: ארמון המגילה.

### 7. שבועות: הר סיני
- Setting: הר לבן-ירוק, ענני סערה, שדות חיטה.
- Visuals: לבן, ירוק, כחול שמים, ברקים.
- Core mechanic: רצפי דיוק, טיפוס, קריאת סימנים.
- Enemies: שומרי פסגה, ענני סערה, סלעים מתגלגלים.
- Boss / finale: סערת ברקים בפסגה.
- Goal set piece: פסגת סיני.

### 8. שמחת תורה: מעגל הסיום
- Setting: עיר ספרים, דגלים, אורות, מעגלי ריקוד.
- Visuals: כחול כהה, לבן, זהב, כתום חגיגי.
- Core mechanic: שילוב מקוצר של כל המכניקות הקודמות.
- Enemies: וריאציות מכל החגים.
- Boss / finale: שומר המעגל.
- Goal set piece: מעגל תורה מואר.

## Design Rules
- כל שלב חייב לקבל:
  - שם חג ברור.
  - subtitle שמתאר את האיום והיעד.
  - palette נפרדת.
  - set piece שונה בסוף השלב.
  - אויבים עם presentation שונה.
  - gimmick שמבדיל אותו משאר השלבים.
- לא מציגים עמים, דמויות דתיות או קהילות חיות כקריקטורה.
- כשאין עדיין art bespoke, משתמשים ב-decoration procedural וב-stage metadata כדי לייצר בידול ראשוני.

## MVP Recommendation
לא לבנות את כל 8 החגים מאפס בפעם אחת.

MVP מומלץ:
- פסח
- חנוכה
- פורים

למה:
- שונות חזקה ברקע ובצבע.
- קל לבנות להם set pieces שונים.
- אפשר להוכיח gameplay variety מהר.

MVP כולל:
- player controller מלא
- collision / physics
- HUD, טיימר, חיים, respawn
- 3 שלבים מלאים
- 3 set pieces שונים לסיום
- 3 שפות ויזואליות שונות
- 1-2 גימיקים stage-level פשוטים

## Current Implementation Strategy
הקוד הקיים הוא בסיס טוב כי כבר יש:
- מנוע canvas
- tilemap
- collisions
- player movement
- HUD
- stage progression
- item/enemy loop

הכיוון המיידי הוא לא לזרוק את הבסיס, אלא להסב אותו:
1. להחליף `stage configs` גנריים ב-stage metadata של חגים.
2. לשנות UI, HUD, title/win copy כך שיספרו "מסע חגים".
3. להוסיף stage-specific backgrounds ו-goal visuals.
4. להוסיף decoration procedural לבלוקים ולאויבים לפי חג.
5. רק אחר כך להעמיק למכניקות ייחודיות אמיתיות לכל חג.

## Execution Plan

### Phase 1: Narrative / Presentation Alignment
- לעדכן `PLAN.md`, `README.md`, `index.html`.
- להחליף `1-1`, `1-2` וכו' בשמות חגים.
- להציג subtitle ספציפי לכל שלב.
- לעדכן מסכי פתיחה/ניצחון לטקסט שמתאר מסע בין חגים.

### Phase 2: Stage Identity Pass
- להוסיף `STAGE_CONFIGS` עם metadata לכל חג.
- להוסיף goal visuals נפרדים לכל חג.
- להוסיף stage atmosphere overlays: פירמידות, שופר, מנורה, עצים, סוכה, מסכות, ברקים, סרטי תורה.
- להוסיף tile decoration ו-enemy adornments per stage.

### Phase 3: MVP Mechanics
- פסח: לבני מצה מתפרקות / מקטעי בריחה מהירים.
- חנוכה: אור שפותח נתיבים.
- פורים: בלבול / מסלולים מתהפכים.

### Phase 4: Scale Up
- להוסיף ט"ו בשבט, סוכות, שבועות, ראש השנה, שמחת תורה כמכניקות מלאות.
- world map ייעודית.
- art pack ייעודי לכל חג.

## Review Focus
Any review of the implementation should ask:
- Does the game read as “Jewish holidays platformer” within the first 10 seconds?
- Does each stage change background, mood and end-goal clearly?
- Do enemy presentation and text support the holiday theme?
- Is the project still accidentally presenting itself as a generic Mario clone?
- Are we shipping a believable MVP path, or pretending the whole campaign is already built?
